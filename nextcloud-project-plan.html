Architecture

VPC / IGW │ ├── EC2 (from Packer AMI) ── IAM instance role │ │ │ ├── RDS MySQL (app database) │ └── S3 bucket (primary file storage) │ └── Elastic IP ── DNS/TLS (deferred — no domain yet) terraform apply (main/) + Packer rebuild — run manually/locally

Terraform is split into two root modules with separate state files — bootstrap/ (password + SSM parameter, applied once) and main/ (everything else, reads bootstrap's outputs via data sources). State backend is S3 only, SSE-KMS, versioning on — no DynamoDB lock table. No CI/CD pipeline — Jenkins was descoped in favor of self-managed applies.

Phase Details

01Networking
Done
  • VPC with public subnet, non-overlapping CIDR
  • Internet Gateway + route table
  • Security group: 443/80 open, 22 restricted to admin IP
  • Elastic IP allocated
  • S3 gateway endpoint
02IAM
Done
  • IAM role for EC2 scoped to the Nextcloud S3 bucket (Get/Put/Delete/List)
  • Instance profile attaching the role
03Database
Done
  • RDS MySQL instance + DB subnet group (private subnets)
  • Security group: 3306 inbound restricted to EC2 SG only
  • Password read via data "aws_ssm_parameter" — main stack never owns the secret
  • Outputs: RDS endpoint, for use in Phase 7
04Golden AMI
Done
  • Base: Amazon Linux 2023, selected via data "aws_ami"
  • Packer builds an EBS-backed AMI, shared to specified accounts/regions
  • Provisions nginx, PHP 8.2 (gd, mbstring, xml, curl, zip, intl, mysqlnd, bcmath, gmp, opcache), Nextcloud, certbot (binary only)
  • nginx configured for PHP-FPM via Unix socket; services enabled, not started at bake time
Open: confirm the component = "clixx" Packer variable is intentional or leftover from a template.
05Compute
Written up
  • data "aws_ami" lookup — owner: self, filtered by name pattern ami-nextcloud-*
  • EC2 instance/launch template: public subnet, instance profile, Phase 1 SG
  • EBS root volume sized for OS + app only — files live in S3
  • Elastic IP association
Open: Phase 7 install via user-data on first boot, or manual SSH?
06DNS & TLS
Deferred

  • Interim: access Nextcloud via the AWS-assigned public DNS name over plain HTTP (port 80)
  • ACM not usable without adding a load balancer — out of scope for now
